# Архитектура NexTalk

## Модули
- client
- server
- common
- ai_service

## Идея
Сервер является ядром системы.
Клиент отображает интерфейс и отправляет команды.
AI-service выполняет анализ файлов по запросу сервера.

## Текущий MVP

На первом этапе реализовано консольное клиент-серверное ядро:

- `server` слушает TCP-порт и принимает подключения;
- на каждого клиента создается отдельный поток;
- пользователь логинится под уникальным именем;
- команда `/msg <user> <text>` отправляет личное сообщение другому онлайн-пользователю;
- сервер пишет события и сообщения в лог-файл;
- `client` использует отдельный поток для приема входящих сообщений.

Локальный режим запускается через `127.0.0.1`.
Сетевой режим запускается подключением клиента к IP-адресу машины с сервером.
