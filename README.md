# NexTalk

Клиент-серверный мессенджер на C++ с отдельным Python AI-сервисом для будущего
анализа вложений.

## Что уже работает

- TCP-сервер на POSIX-сокетах.
- Консольный TCP-клиент.
- Логин пользователя.
- Личные сообщения между двумя онлайн-пользователями.
- Отдельный поток клиента для приема входящих сообщений.
- Отдельный поток сервера на каждого подключенного клиента.
- Логирование старта/остановки сервера, входов, подключений, отключений и сообщений.
- Запуск в локальном режиме через `127.0.0.1` и в сетевом режиме через IP сервера.

## Структура проекта

- `client/` - клиентская часть на C++.
- `server/` - серверная часть на C++.
- `common/` - общий протокол и общие структуры.
- `ai_service/` - AI-сервис на Python.
- `docs/` - документация по архитектуре и протоколу.

## Сборка

```bash
cmake -S . -B build
cmake --build build
```

## Запуск

Сервер:

```bash
./cmake-build-debug/server/server 5555 0.0.0.0 server.log
```

Клиент на той же машине:

```bash
./cmake-build-debug/client/client 127.0.0.1 5555

```

Клиент с другой машины или виртуалки:

```bash
./cmake-build-debug/client/client 127.0.0.1 5555
```

Команды клиента:

```text
/msg <user> <text>  отправить личное сообщение
/login <user>       сменить логин
/help               показать команды
/quit               выйти
```

Для проверки можно открыть три терминала: один для сервера и два для клиентов.
Первый клиент логинится как `alice`, второй как `bob`, затем `alice` отправляет:

```text
/msg bob Привет
```
